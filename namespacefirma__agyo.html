Descrizione dettagliata

Il modulo è un wrapper sviluppato per accedere ai servizi di firma elettronica offerti da Agyo.

Le funzionalità vengono richiamate dai programmi COBOL che le alimentano tramite dei file di input (dacbl) con record a campi di posizione e dimensione fissa. Ogni funzionalità scrive i propri risultati in file di output (acbl) con record a campi di posizione e dimensione fissa

Gestisce il download di un documento già caricato in Agyo (hubid presente e stato Firmato).

Prima di scaricare il file nè viene valutato lo stato e si procede al download solo se lo stato è = Firmato. Se lo stato del file è diverso da Firmato viene scritto nel file acbl al posto del nome file Il risultato dell'operazione viene scritto nei file di output acbl.

Parametri
[in]WSPAR_TOKEN, str, token di accesso ad Agyo
[in]WSPAR_APP, str, Nome applicazione chiamante verso Agyo
[in]WSPAR_INP, str, path file di input (dacbl)
[in]WSPAR_TRANSMITTER_ID, str, transmitter_id (codice fiscale ?)
[in]WSPAR_SENDER_ID, str, sender_id (codice fiscale seguito da -000?)
[in]WSPAR_CONFIG, str, path del file di configurazione (config.xml)
[in]WSPAR_PROXY_URL, str, url dell'eventuale proxy
[in]WSPAR_PROXY_PORT, str, porta di ascolto dell'eventuale proxy
[in]WSPAR_TIMEOUT, str, timeout della connessione verso il servizio
[in]WSPAR_PROC_NAME, str, nome del programma (COBOL) chiamante es : 'MULTI'
[in]WSPAR_EXTRAPATH, str, percorso aggiuntivo dal file chiamante al file firma_agyo.py. Questa extra path viene richiesta quando la chiamata alle funzionalità del modulo firma_agyo è delegata, cioè quando firma_agyo viene richiamato da un altro modulo che non si trova nella stessa directory di firma_agyo esempio: wslynfa.py
